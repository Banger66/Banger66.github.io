Explanation of control flow and loops

JavaScript uses methods to make sure everything works uniformly. Control Flow refers to the computer running code from top to bottom, and loops refer to doing a patch of code over and over again until a certain condition is met. Think of it like making a bowl of soup. You follow the recipe from top to bottom, but sometimes you are required to perform the same task over and over again, like stirring the pot X amount of times until all the ingredients are mixed together properly and the soup is consistant.

What is the DOM?

DOM stands for Document Object Model. In the world of web, all HTML webpages are referred to as documents. The Document Object Model represents each of these web pages in a tree-like structure to make it easier to access and manage the elements. The DOM's contents are literally everything inside a webpage, and sometimes we might want to interact with that using JavaScript. For example, when you go to log in or sign up to a website, you enter some credentials and send them using a button. This would be a good example of interacting with the DOM.

The Difference between accessing data from arrays and objects

Objects are mutable data structure in javascript which is used to represent a ‘Thing’. This could be anything like cars, plants, person, community etc. Objects in JavaScript are displayed like this:

        
   let harry = {
      name : 'Harrison Mordecai';
      age : '26';
      gender : 'male' ;
      favHobby : 'Dota 2'; 
   }

It stores the data in what we call key value pairs, for example the key name is paired with it's value, 'Harrison Mordecai'. We can either use a dot operator followed by key name obj.name or a square bracket along with key in string format obj['name'] to access the value.

Now, onto arrays. The major difference is that they store the data in an ordered collection in which the data can be accessed using a numerical index. Here's an example of an array:

      
        let agesInHouseHold = [26, 23, 2, 0]

Spans were the first thing to make me realize why programmers count from 0. you can see 4 pieces of data in the array, so you would think, in order they would be listed as 1-2-3-4-, right? Nuh uh. Computers are so smart, but holy crap, so dumb too. They count from 0, so referring back to our array, they're listed as 0-1-2-3. It's already caught me out a few times and I'm sure it's gonna get me a lot more in the future

Functions

Functions are the bread and butter of JavaScript. It's like a procedure - a set of statements that performs a task or calculates a value, but for a procedure to qualify as a function, it should take some input and return an output where there is some obvious relationship between the input and the output. Here's a super simple example:

      
        function myFunction(p1, p2) {
          return p1 * p2;   
        }

In this function, called 'myFunction', it's taking two variables and multiplying them together. The beauty of using functions is that we can reuse the same code over as many times as we want. So no matter what happens to the variables p1 and p2, we can call this function again to multiply them, no matter what their value is.
